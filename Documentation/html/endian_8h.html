Detailed Description

Allows us to map to/from bytes stored in little endian order (irrespective of host machine endianess).

Also as a side-effect, it allows out of alignment access.

And another side-effect is that structures of these will be tightly packed, there is no padding as would happen with the normal uint16_t and uint32_t. The padding is added for the reason of alignment, but because these types can be accessed without any particular byte alignment, then padding isn't needed.
