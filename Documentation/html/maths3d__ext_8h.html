Detailed Description

Extensions to the Maths3D API.

Transforming large arrays of vectors is performance critical to many 3D pipelines. This file contains SSE optimizations for doing this in a flexible way that this may be useful in a variety of situations. However this is platform specific code, so fallback code may be required.

It should be a todo that based on a preprocessor define, this SSE optimized code is used or some fallback code is used instead which would need to be written.

Transforms arrays of vectors by the transform matrix.

Template Parameters
translateis a bool to enable or disable applying the translation component of the transform.
divideByWis a bool to enable or disable applying perspective by dividing by W.
alignedOutputis a bool to flag if outputStream is aligned to a 128-bit boundary or not.
outputStepis the width in floats to the next vector element of the outputStream array. This allows the outputStream array to be an array of structures of which the vector is a member. These structures should be 32-bit aligned, and if the size is not a multiple of 128-bits, then alignedOutput shouldn't be set.
alignedInputis a bool to flag if inputStream is aligned to a 128-bit boundary or not.
inputStepis the width in floats to the next vector element of the inputStream array. This allows the inputStream array to be an array of structures of which the vector is a member. These structures should be 32-bit aligned, and if the size is not a multiple of 128-bits, then alignedInput shouldn't be set.
Parameters
outputStreamis the output buffer to put the transformed vectors to.
inputStreamis the input array of vectors to apply the transform to.
countis the number of vectors to transform.
transformis the matrix to apply.
